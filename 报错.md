【修改点 2（截图中的 2）：短线要与日期刻度完全对齐】
------------------------------------------------

问题：现在从时间轴往上伸出的“短线”，有时与下面轴上的日期刻度对不齐，看起来有点偏。

目标：每一列 `.timeline-day-column` 对应的竖线必须与它那天的时间轴刻度 **同一个 x 坐标**，也就是说：
- 轴上的日期刻度位置（比如 2025-11-11）有一个小圆点 / 刻度；
- 从这个点**正上方**伸出一根竖线；
- 竖线正上方是该日期的事件卡片列（中心对齐）。

实现思路（请在 timeline.js 里改）：

1. 在 d3-milestones 渲染完成后（decorateMilestones 或 renderTimeline 内部）：
   - 遍历所有 `.milestones__group`：

     - 同时通过绑定的数据，得到这一天的 dayKey（例如用我们已有的 formatDayLabel(timestamp) 得到 "2025-11-11"）。
   - 把 `dayKey -> x` 存为一个 map，例如：
     - `const dayXMap = { '2025-11-11': 120, '2025-11-13': 260, ... };`
     - 可以挂在一个全局对象上，或者闭包中，但要能在 renderTimelineList 中访问。

2. 在 `renderTimelineList(entries)` 中，渲染 `.timeline-day-column` 的时候：
   - 现在每一列是按 groupedByDay 顺序生成的；
   - 对每一列 `([dayKey, events])`：
     - 从 `dayXMap[dayKey]` 拿到对应的 x 坐标；
     - 使用 absolute 定位或 transform，使这一列的中心对齐到这个 x：
       - 比如：
         - 给 `#health-timeline-list` 设置 `position: relative;`
         - 每个 `.timeline-day-column`：
           - `position: absolute;`
           - `left: dayX - columnWidth / 2`  或  使用 `transform: translateX(dayX - columnWidth / 2)`
   - 同时，竖线的 x 坐标也要用同一个 dayX：
     - 如果竖线是 SVG 里的 `<line class="health-axis-tick">`，那它的 x1/x2 应该等于对应 group 的 x；

【修改点 3（截图中的 3）：时间轴日期要连续显示，不跳过】
------------------------------------------------

问题：现在时间轴底部只展示有事件的日期，比如 2025-11-11、2025-11-13、2025-11-15，中间的日期（例如 12 号、14 号）被跳过，轴上看起来不连续。

目标：时间轴底部的日期刻度要 **按天连续显示**，即：
- 从整个数据区间的最小日期到最大日期，中间**每一天都有刻度和日期文字**；
- 即使当天没有任何事件，也要在轴上保留刻度和日期（列可以为空）。

实现思路（在 timeline.js 中改）：

1. 在构建时间轴数据时，先得到时间范围：
   - 从所有事件 entries 中取出最小日期 `minDate` 和最大日期 `maxDate`（可以只看日期不看具体时间）；
2. 使用 d3 生成完整的日期序列：
   - `const allDays = d3.timeDay.range(minDate, d3.timeDay.offset(maxDate, 1));`
   - 这样 allDays 就包含从 minDate 到 maxDate（含）的每一天。

3. 更新 d3-milestones 或自定义轴刻度逻辑：
   - 如果你用的是 d3 自己画 axis，则让 xScale 的 tickValues = allDays；
   - 如果用 d3-milestones 默认刻度，可以在 decorateMilestones 之后额外画一条自定义的日期轴：
     - 在 #health-timeline-chart 的 SVG 里单独画一组 `<g class="health-date-axis">`；
     - 遍历 allDays：
       - 对每个 day 计算 x = 同一套时间 scale(day)；
       - 在该 x 上画刻度线和日期文字（格式为 "YYYY-MM-DD"）；
   - 重要的是：**即使某一天没有事件 entry，也要画出刻度与日期文字**。

4. 与第二点的 dayXMap 保持一致：
   - 当你计算 allDays 时，确保 dayXMap 里使用的 x 与这条连续日期轴使用的是同一套 scale 和日期格式；
   - 这样列和刻度就不会错位。
